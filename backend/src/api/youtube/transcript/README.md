# YouTube Transcript API

Fetches English (or any requested language) transcripts for a YouTube video via the iOS InnerTube `/player` endpoint + timedtext JSON3.

## Endpoint

| Method | Route | Config key |
|--------|-------|------------|
| `POST` | `/youtube/video/transcript` | `API_ENDPOINTS.YOUTUBE.VIDEO_TRANSCRIPT` |

## Request body

`schemas.ts` → `YOUTUBE_VIDEO_TRANSCRIPT_REQUEST_SCHEMA` (extends `YOUTUBE_GEO_REQUEST_SCHEMA`):

| Field | Type | Default | Notes |
|-------|------|---------|-------|
| `country` | `string` | `"US"` | 2-letter ISO — proxy + InnerTube `gl` |
| `region` | `string` | — | Optional proxy region |
| `videoId` | `string` | required | Alphanumeric, max 20 |
| `language` | `string` | `"en"` | BCP-47 code (e.g. `en`, `es`, `en-US`) |

## Response

`ALApiResponse<YOUTUBE_VIDEO_TRANSCRIPT_RESPONSE>`:

```ts
{
  videoId: string;
  language: string;              // selected track's BCP-47 code
  isAutoGenerated: boolean;      // true if selected track is ASR
  availableTracks: YOUTUBE_TRANSCRIPT_TRACK[];
  lines: YOUTUBE_TRANSCRIPT_LINE[];
  fullText: string;              // lines joined with spaces
}
```

## How it works

As of 2026, the WEB and ANDROID InnerTube clients require BotGuard/PO-Token
attestation for transcript-related requests, which cannot be generated
server-side without executing YouTube's obfuscated JavaScript. The **iOS
client is exempt** from this requirement, making it the reliable server-side
path.

Strategy:

1. **`POST /youtubei/v1/player`** with the iOS InnerTube client (`YOUTUBE_INNERTUBE_IOS_CLIENT`)
   → returns `captions.playerCaptionsTracklistRenderer.captionTracks[]`, each
   with a per-language `baseUrl`.
2. **`GET baseUrl + &fmt=json3`** → fetches the timedtext JSON3 payload.
3. **Flatten** `events[].segs[].utf8` into ordered transcript lines.

Language selection preference (`pickTrack`):

1. Exact language code, non-ASR (manually authored).
2. Exact language code, any kind.
3. Language prefix match (e.g. `en-US` for requested `en`), non-ASR.
4. Language prefix match, any kind.
5. First available track.

## DRY rules

| Concern | Where it lives | Notes |
|--------|---------------|-------|
| Track kind `"asr"` | `constants.ts` → `YOUTUBE_TRANSCRIPT_TRACK_KIND` | Used in `compute.ts` + `helpers.ts` |
| timedtext `fmt=json3` | `constants.ts` → `YOUTUBE_TIMEDTEXT_FORMAT` | Used in `fetch.ts` |
| `fmt=` strip regex | `constants.ts` → `YOUTUBE_TIMEDTEXT_FMT_PARAM_PATTERN` | Used in `compute.stripFmtParam` |
| Playability OK statuses | `constants.ts` → `YOUTUBE_PLAYER_PLAYABILITY_OK_STATUSES` | Used in `fetch.ts` |
| Default `hl` | `constants.ts` → `YOUTUBE_TRANSCRIPT_DEFAULT_HL` | Used in `schemas.ts` + `fetch.ts` |
| InnerTube/JSON3 raw shapes | `types.ts` | Single source of truth for internal types |
| Pure track/line logic | `compute.ts` | No I/O, no side effects |
| HTTP I/O | `fetch.ts` | InnerTube POST + timedtext GET |
| Orchestration | `helpers.ts` | Session lifecycle + composition only |
| iOS client config | `../constants.ts` → `YOUTUBE_INNERTUBE_IOS_CLIENT` | Shared across the YouTube module |

## Notes for agents

- The iOS `baseUrl` is **not** PoT-gated, so it can be fetched server-side
  without executing YouTube's player JavaScript.
- Do not switch to the WEB or ANDROID client for transcript requests —
  both return `400 FAILED_PRECONDITION` as of 2026.
- The deprecated `get_transcript` InnerTube endpoint is not used; it
  requires BotGuard attestation that cannot be generated server-side.
